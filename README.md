# Проект: Тестирование приложения для зоопарка (семейство кошачьих)

Проект по автоматизации тестирования Java-приложения, моделирующего иерархию и поведение кошачьих. Основной фокус сделан на модульном тестировании, изоляции зависимостей и обеспечении полного покрытия кода.

## 🛠 Стек технологий
* **Java 11**
* **JUnit 4** — написание модульных и параметризованных тестов.
* **Mockito** — создание моков (заглушек) и проверка взаимодействия объектов.
* **Jacoco** — инструмент для оценки покрытия кода тестами.
* **Maven** — сборка проекта и управление зависимостями.

## 🎯 Ключевые задачи и решения
1. **Рефакторинг и DI (Dependency Injection):** В классе `Lion` была устранена жесткая зависимость от класса `Feline`. Теперь объект `Feline` передается через конструктор, что соответствует принципам **SOLID** и упрощает тестирование.
2. **Изоляция тестов:** С помощью **Mockito** были созданы моки для внешних зависимостей. Это позволило тестировать логику конкретного класса (например, `Cat` или `Lion`) независимо от работы других компонентов.
3. **Параметризация:** Для проверки логики наличия гривы у льва использованы параметризованные тесты, что позволило одной функцией проверить разные входные данные (самец/самка).
4. **Дополнительный функционал:** Реализован класс `Alex`с уникальными характеристиками, переопределением логики родительского класса и собственным набором тестов.

## 💻 Демонстрация (Main)
В проекте реализован демонстрационный класс `Main`, который имитирует работу приложения:
* Создание объектов разных типов кошачьих.
* Вызов методов поведения (голос, питание, количество потомства).
* Работа с уникальным объектом льва Алекса.

## 🚀 Как запустить тесты
Для запуска тестов и генерации отчета Jacoco выполните команду:
mvn clean verify
